# PRES Prioritization

PRES is the **single score at the opportunity level**. It replaces a separate "Confidence" rating (they'd say the same thing). At solutions/assumption tests, use Confidence instead (see `confidence-scoring.md`).

## PRES = Pain x Reach x Evidence x Strategic

Each factor 1-5. Sum 4-20. The score is a conversation starter, not truth. Disagreement in the team = gold.

### Pain - how badly does this hurt the user

Not "how much it annoys the team" or "how visible on the roadmap." From the user's perspective.

- **1** - minor inconvenience. User completes the flow, doesn't think about it again.
- **3** - blocks a step. User seeks workarounds (other device, support contact, alternative service).
- **5** - abandonment, return, refund, negative review, churn. User pays with money or time.

### Reach - how many users, how often

Population x frequency. A problem hitting 2% of users daily may outreach one hitting 30% yearly.

- **1** - edge case. <5% of users or less than quarterly.
- **3** - significant group (15-40%) monthly, or smaller group per session.
- **5** - >70% of users every session/purchase. Visible on every dashboard.

Measure from data, not gut feel. If you don't know, Reach goes low until you check.

### Evidence - how strong is our signal that this problem exists as described

The anti-HIPPO checkpoint.

- **1** - team hypothesis, no data.
- **3** - repeating signal. 3+ interviews say the same thing, OR quantitative data points to the problem.
- **5** - triangulation. Interviews + analytics + reviews/tickets all point the same way. 3 independent sources.

**Hard rule:** if Evidence = 1, stop scoring. Go talk to users first.

### Strategic - does solving this move our outcome

A real problem (Pain 5, Reach 5, Evidence 5) that doesn't affect the outcome KPI is not your problem right now.

- **1** - tangential. Solving it won't move the outcome.
- **3** - indirect support. Contributes but isn't the leverage point.
- **5** - direct hit. Solving this moves the outcome KPI significantly.

## Presentation

Show per-axis reasoning with citations. Format: opportunity name, then each axis with score + one-line rationale citing the source. Then PRES total. This makes disagreement productive - user can point at one axis.

After presenting, ask which opportunities to pursue. If two tie within 1 point, say so and let the user's intuition break the tie.

## Always cite

A score without citations is a guess. If the source for an axis doesn't exist, drop that axis to 1-2.
